Summary

Midas is a machine learning project inspired by the Kaggle IEEE-CIS challenge to train a model to predict credit-card fraud. Our project is generalized in that it will import any dataset (formatted as a CSV), provide rudimentary validation, and give the user options for customizing a training session, analyzing the results, and executing saved models against new data.

The Pipeline in Detail

The typical workflow would involve the following steps: importing data, analyzing the data to determine the best approach for cleaning and training, cleaning the data, training the model, and then executing the model on new data to make predictions. In addition, we have provided the user some basic CRUD functionality in that the user may delete past models, save new models, and retrieve those models in deciding which to execute. In new releases, we would ideally provide a more robust interface, in particular letting users restore previous training sessions to make minor changes. The pipeline in detail goes through the following steps:
  • Data Import
    As of right now, our website performs almost no validation of imported files except that they must be in CSV format and be non-empty. Future releases would contain much more robust validation.
  • Data Preparation
    In this step, the user may choose the response variable and then they may confirm suggested data types for features. In effect, with any dataset, we can programmatically infer the datatypes of each feature, but since numbers can represent categories, the user may wish to customize which features containing only numbers are actually categorical. In fact, the Kaggle IEEE-CIS dataset has many features which contain only numeric data but which are actually categorical. After the user chooses the response variable and confirms data types for features, they move on to analysis.
  • Data Analysis
    Our codebase will analyze the chosen dataset programmatically by collecting summary and frequency statistics on features. The user may wish to use this analysis to decide on appropriate data cleaning steps. In addition, each feature will be given a link the user may click to get detailed statistics for each feature, including summary statistics, frequency statistics, and histograms/bar charts.
  • Algorithm Selection
    The user is prompted to first select a machine learning algorithm before doing any data cleaning. This is necessary because the required and optional data cleaning methods may depend on the particular machine learning algorithm. We have implemented four algorithms: K-Nearest Neighbors, Adaboost, Support Vector Machines, and Random Forests.
  • Data Cleaning and Training
    After selecting the machine learning algorithm, a form will be dynamically generated to customize a training session. The user will have some required fields and some optional fields. Once selected, our codebase will then begin training and then displaying a summary of the results.
  • Summary Of Results
    The user will be given some basic analysis of the results, including the mean area under a curve, a confusion matrix, and a list of algorithm parameters. The user is then prompted to give the model a name to save the model to execute later.
  • Importing New Data and Executing Models
    After saving new trained models, the user can then select the dropdown near the top of the page to switch to algorithm execution. The user must upload new data against which to test the model. Naturally, this data must be formatted the same as the training data. Currently, our website does not validate this imported data. The user then can select the training model to run, execute the model and see the results.
